# Engineering Domain Axioms

## AX-DOM-ENGINEERING-001: Test Everything

Every new feature or fix should include tests that validate the behavior. Tests should cover:
- The happy path
- Edge cases
- Failure modes

## AX-DOM-ENGINEERING-002: Code Review Before Merge

All non-trivial changes should be reviewed before merging to main.

## AX-DOM-ENGINEERING-003: Keep It Simple

Prefer simple solutions over complex ones. Complexity is a liability.
