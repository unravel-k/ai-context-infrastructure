# Risk Analysis Axioms

See `.agentic/axioms/skills/risk-analysis.md` for the canonical axiom definitions.

## AX-SKILL-RISK-ANALYSIS-001: Impact Before Likelihood

Assess impact first, then likelihood. This prevents the anchoring effect where a high-likelihood view gets inflated impact, or a low-likelihood view gets dismissed regardless of its potential magnitude.

## AX-SKILL-RISK-ANALYSIS-002: Direct Reasoning Only

Every "why" explanation must be a direct reason about the side or view being discussed. Do not explain Side A's chances by referencing Side B's weakness. "Side A is strong because X" — not "Side A is better because Side B is weak."

## AX-SKILL-RISK-ANALYSIS-003: No Pseudo-Quantitative Precision

Use qualitative labels (Critical, High, Medium, Low, Negligible). Do not assign fake percentages (e.g., "67.3% chance"). Qualitative labels are honest about uncertainty; invented numbers create false confidence.

## AX-SKILL-RISK-ANALYSIS-004: Source Preservation

Every view evaluated must trace back to its original source. If consuming for-and-against output, preserve all attribution links. If a view arrives unattributed, flag it clearly.
