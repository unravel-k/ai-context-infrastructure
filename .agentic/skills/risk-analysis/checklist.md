# Risk Analysis Checklist

- [ ] Received input views (from for-and-against output, user, or other source)
- [ ] Identified all distinct sides of the bet or decision
- [ ] Extracted individual views per side with sources preserved
- [ ] Evaluated impact for each view (qualitative label + why)
- [ ] Evaluated likelihood for each view (qualitative label + why)
- [ ] Verified every "why" is a direct reason about that side (not comparison)
- [ ] Built risk profile with aggregate impact, likelihood, chance of success, key risk
- [ ] Used qualitative labels only (no fake percentages)
- [ ] Wrote qualitative summary stating which side has better chances and why
- [ ] Preserved source attribution throughout
