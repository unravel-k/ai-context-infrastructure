# For and Against Axioms

See `.agentic/axioms/skills/for-and-against.md` for the canonical axiom definitions.

## AX-SKILL-FOR-AND-AGAINST-001: Separate Queries

Always query opposing directions independently. Never combine "for" and "against" into a single query — this introduces framing bias.

## AX-SKILL-FOR-AND-AGAINST-002: Tools First

Check for and use available specialized tools (MCP, data APIs) before falling back to general web search. Specialized tools produce higher-quality, more structured data.

## AX-SKILL-FOR-AND-AGAINST-003: Full Attribution

Every claim must carry a source attribution. Use URL + title, or title + author, whichever is available. Never present an unattributed claim.

## AX-SKILL-FOR-AND-AGAINST-004: Consensus vs Contrarian

When multiple views exist on the same sub-topic, classify each as consensus (majority view) or contrarian (minority view). If no majority exists, state that views are mixed.

## AX-SKILL-FOR-AND-AGAINST-005: No Fabricated Balance

Do not invent equal weight when evidence leans strongly one way. Report the imbalance honestly rather than manufacturing false balance.
